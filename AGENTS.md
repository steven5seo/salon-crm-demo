# Repository Guidelines

- Use semantic, accessible HTML. Provide ARIA roles only when necessary.
- Keep indentation at two spaces for HTML, CSS, and JavaScript.
- Favor descriptive naming for files and selectors.
- Do not introduce build tooling; keep assets static.
- Document any new pages in `README.md` when adding them.
- Keep commit messages in the imperative mood.
- When updating structured content (JSON data files), ensure the format stays valid and sorted when practical.
